start/1

start(Config::network_config()) -> ok | {error, Reason::term()}

Config: The network configuration

returns: ok, if the network_fsm was started, or {error, Reason} if a failure occurred (e.g., due to malformed network configuration).

Start a network_fsm.

This function will start a network_fsm, which will attempt to connect to an AP endpoint in the background. Specify callback functions to receive definitive information that the connection succeeded. See the AtomVM Network FSM Programming Manual for more information.

Start a network_fsm in access point mode and wait the AP to be up and running

This function will start a network_fsm in AP mode, and will wait until the network is up and ready to be connected. This is a convenience function, for applications that do not need to be notified of connectivity changes in the network.
